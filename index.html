Transformer Layer Roofline Analysis

This interactive page helps you understand the computation, memory IO, and hardware utilization of each layer in a modern LLM, using OpenAI's GPT-OSS-20B (a 20.9B-parameter sparse MoE model) as a running example. Use the sliders to explore how sequence length, model dimensions, and batch size affect whether each layer is memory-bound or compute-bound on an NVIDIA A100 GPU.
The roofline model bounds attainable performance by the minimum of peak compute and memory bandwidth. For computational intensity $\text{CI} = \frac{\text{FLOPs}}{\text{Bytes}}$, the attainable performance is $\text{Perf} = \min(\text{Peak FLOPS},\; \text{CI} \times \text{BW})$. The ridge point $\text{CI}^* = \frac{\text{Peak}}{\text{BW}}$ separates memory-bound (left) from compute-bound (right) regimes.
BF16 peak: 312 TFLOPS HBM2e BW: 2.0 TB/s Ridge CI: 156 FLOP/B

Session 1: Transformer Architecture

RMSNorm
RMSNorm normalizes each token by its root-mean-square, improving training stability without centering. $$\text{RMS}(\mathbf{x}) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2}, \qquad \text{RMSNorm}(\mathbf{x}) = \boldsymbol{\gamma} \odot \frac{\mathbf{x}}{\text{RMS}(\mathbf{x}) + \epsilon}$$ where $\boldsymbol{\gamma} \in \mathbb{R}^d$ is a learnable scale. With CI $\approx 1$ FLOP/byte, this is always deeply memory-bound.
RMSNorm(x) = γ ⊙ x / (RMS(x) + ε)   where   RMS(x) = √(1/d × Σ x_i²)
FLOPs = S × d × 4   (square, accumulate, div-by-RMS, scale-by-γ per element)
IO = (2×S×d + d) × 2 bytes

Grouped-Query Attention (naive, without Flash-Attention)
Grouped-Query Attention (GQA) reduces KV heads while keeping many query heads. $H$ query heads share $H_\text{kv}$ key-value heads (group size $G = H/H_\text{kv}$). $$\mathbf{Q} = \mathbf{X}\mathbf{W}_Q,\quad \mathbf{K} = \mathbf{X}\mathbf{W}_K,\quad \mathbf{V} = \mathbf{X}\mathbf{W}_V$$ $$\mathbf{O}_h = \text{softmax}\!\left(\frac{\mathbf{Q}_h \mathbf{K}_{h'}^\top}{\sqrt{d_h}}\right)\mathbf{V}_{h'}, \qquad \mathbf{O} = \text{Concat}(\mathbf{O}_1,\ldots,\mathbf{O}_H)\,\mathbf{W}_O$$ In the naive (non-Flash) prefill, the full $S \times S$ attention matrix is materialized to HBM per head.
Materializes full S x S attention matrix to HBM per head — includes O_proj
FLOPs = 2×S×d×hd×(2H+2H_kv) + H×S²×(4×hd+5)
IO_proj = [2×S×d + d×hd×(2H+2H_kv) + S×hd×(2H+2H_kv)] × 2
IO_attn = H × [4×S×hd + 4×S²] × 2

Dense FFN (SwiGLU)
SwiGLU FFN uses a gated linear unit with Swish activation. The gate projection $\mathbf{W}_1$ and up projection $\mathbf{W}_3$ produce two $d_\text{ff}$-dimensional vectors combined element-wise, then the down projection $\mathbf{W}_2$ maps back to $\mathbb{R}^d$. $$\text{FFN}(\mathbf{x}) = \bigl(\text{Swish}(\mathbf{x}\mathbf{W}_1) \odot \mathbf{x}\mathbf{W}_3\bigr)\mathbf{W}_2, \qquad \text{Swish}(z) = z \times \sigma(z)$$
FFN(x) = (Swish(xW₁) ⊙ xW₃) W₂
FLOPs = S × d_ff × (6×d + 6)  |  IO = d × d_ff × 1.59 + S × (6×d + 12×d_ff) bytes

Mixture of Experts (MoE)
Mixture of Experts (MoE) sparsely scales model capacity by routing each token to the top-$K$ of $E$ expert FFNs. The router computes scores $\mathbf{g} = \text{softmax}(\mathbf{W}_r \mathbf{x})$, selects the top-$K$ experts $\mathcal{T}$, and the output is: $$\text{MoE}(\mathbf{x}) = \sum_{i \in \mathcal{T}} \hat{g}_i \times \text{FFN}_i(\mathbf{x}), \qquad \hat{g}_i = \frac{g_i}{\sum_{j \in \mathcal{T}} g_j}$$ Weight IO is dominated by $K_\text{unique}$ (unique experts activated across the batch).
MoE(x) = Σ p_i × FFN_i(x)
FLOPs = S×d×E×2 + K×S×d_ff×(6×d+6) + S×d×(2×K−1)
IO = (S×d + S×E + d×E)×2 + K_u×d×d_ff×1.59 + K×S×(6×d+12×d_ff) + S×d×(2×K+2)×2 bytes

LM Head (Linear + Softmax)
The LM Head maps the final hidden state to vocabulary-sized logits and applies softmax to produce next-token probabilities. $$\text{logits} = \mathbf{x}\,\mathbf{W}_\text{lm}, \qquad p_i = \frac{\exp(\text{logit}_i)}{\sum_j \exp(\text{logit}_j)}$$ where $\mathbf{W}_\text{lm} \in \mathbb{R}^{d \times V}$ with $V$ = vocabulary size (201,088 for GPT-OSS-20B).
logits = x × W_lm   then   p = softmax(logits)
FLOPs = S×d×V×2 + S×V×4
IO = (S×d + d×V + S×V×3) × 2 bytes   (write logits + read for softmax + write probs)

GQA Decode (1 token + KV cache, naive, with O_proj)
During decode, a single new token $\mathbf{x}_t$ is projected to $\mathbf{q}_t$, $\mathbf{k}_t$, $\mathbf{v}_t$. The new $\mathbf{k}_t, \mathbf{v}_t$ are appended to the KV cache, then attention is computed against all $S_\text{KV}$ cached entries: $$\mathbf{o}_t = \sum_{i=1}^{t} \text{softmax}\!\left(\frac{\mathbf{q}_t^\top \mathbf{k}_i}{\sqrt{d_h}}\right) \mathbf{v}_i$$ This is deeply memory-bound: very little compute (no $S^2$ matmul) but the entire KV cache must be read from HBM.
Single new token attending to S_KV cached tokens — no Flash-Attention
FLOPs = 2×d×hd×(2H+2H_kv) + H×S_KV×(4×hd+5)
IO_proj = [2×d + d×hd×(2H+2H_kv) + hd×(2H+2H_kv)] × 2
IO_attn = H × [2×hd×(S_KV+1) + 4×S_KV] × 2

Session 2: Flash Attention

Flash Attention vs Naive Attention (Prefill)
Flash Attention computes exact attention without materializing the $S \times S$ attention matrix to HBM. By tiling the computation and keeping the attention matrix in SRAM, the HBM IO reduces to just reading Q, K, V and writing O — eliminating the $O(S^2)$ memory traffic. $$\text{IO}_\text{naive} = H \times (4 S \cdot hd + 4 S^2) \times 2 \qquad \text{IO}_\text{flash} = H \times 4 S \cdot hd \times 2$$ FLOPs remain identical — Flash Attention changes only the memory access pattern, not the computation.
FLOPs (both) = H × S² × (4×hd + 5)
Naive IO = H × (4×S×hd + 4×S²) × 2   (materializes S×S attn matrix)
Flash IO = H × 4×S×hd × 2   (only Q, K, V, O to HBM)
